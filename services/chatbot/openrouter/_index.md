## 🎯 服务简介

OpenRouter Playground 是一个强大的 Web 测试界面，让您可以直接在浏览器中测试和对比 47+ 个免费 AI 模型，无需编写任何代码。

**核心优势：**
- 🎯 **47+ 免费模型** - 一个界面测试所有模型
- 🔄 **模型对比** - 同时测试多个模型
- 📊 **使用统计** - 实时查看配额和成本
- 🎛️ **参数调节** - 自定义生成参数
- 💾 **历史记录** - 保存和管理对话

## 🚀 如何使用

### 前提条件

- ✅ 已注册 OpenRouter 账户
- ✅ 已创建 API 密钥（Playground 会自动使用）

详细注册步骤请参考：[OpenRouter 注册指南](/providers/openrouter#注册和账号)

### 使用步骤

#### 步骤 1：访问 Playground

1. 访问：https://openrouter.ai/playground
2. 使用账户登录（会自动识别）

#### 步骤 2：选择模型

在模型下拉菜单中选择免费模型（标记为 **FREE**）：

**推荐模型：**

| 模型 | 适用场景 | 特点 |
|------|---------|------|
| **Llama 3.3 70B** | 通用任务 | Meta 最新，性能强 |
| **Llama 3.1 405B** | 复杂任务 | 超大参数，能力顶尖 |
| **DeepSeek R1** | 推理任务 | 数学、代码专家 |
| **Qwen 3 235B** | 中文任务 | 阿里通义千问旗舰 |
| **Mistral 7B** | 快速响应 | 轻量高效 |

#### 步骤 3：输入提示词

1. 在输入框中输入您的问题或提示词
2. 可以使用 System Message 设置角色
3. 点击发送或按 Enter 键

#### 步骤 4：调整参数（可选）

在右侧面板调整生成参数：

| 参数 | 范围 | 说明 | 推荐值 |
|------|------|------|--------|
| **Temperature** | 0-2 | 控制创造性 | 0.7 |
| **Max Tokens** | 1-无限 | 限制输出长度 | 1024 |
| **Top P** | 0-1 | 核采样概率 | 0.9 |
| **Top K** | 0-无限 | 候选词数量 | 40 |
| **Frequency Penalty** | -2 to 2 | 降低重复 | 0 |
| **Presence Penalty** | -2 to 2 | 增加话题多样性 | 0 |

### 1. 多模型选择

**免费模型分类：**

- **🏆 顶级大模型：** Llama 3.3/3.1 405B, DeepSeek R1
- **🚀 高性能模型：** Qwen 3 235B, Mistral Small, Gemma 3 27B

**功能：**
- 定义 AI 的角色和行为

**示例：**

**功能：**
- 📝 **保存对话** - 重要对话可保存

### 4. 使用统计

**实时显示：**
- 💰 **成本统计** - 查看每次请求成本（免费模型显示 $0）

- 📈 **配额使用** - 当日剩余请求数
- ⏱️ **响应时间** - 请求耗时

### 5. 模型对比模式

**功能：**
- 同时运行多个模型
- 对比输出质量
- 比较响应速度
- 选择最适合的模型

**使用场景：**
- 选择最佳模型

- 评估模型性能

**功能：**

- 🔗 **生成链接** - 分享对话

## 💡 使用技巧

### ✅ 最佳实践

1. **选择合适的模型**
   ```
   快速测试 → Mistral 7B, Llama 3.2 3B
   通用任务 → Llama 3.3 70B
   复杂任务 → Llama 3.1 405B
   中文优化 → Qwen 3 235B, DeepSeek V3
   代码任务 → Devstral, Qwen 3 Coder
   数学推理 → DeepSeek R1
   ```

2. **优化 System Prompt**
   - 明确角色和任务
   - 提供具体要求
   - 给出输出示例

3. **利用对比功能**
   - 测试相同 Prompt 在不同模型上的表现
   - 找到最适合您任务的模型
   - 记录最佳配置

4. **监控配额**
   - 在右上角查看剩余配额
   - 合理分配测试次数
   - 考虑升级到 1000 次/天

5. **管理对话历史**
   - 保存重要的测试结果
   - 定期清理不需要的历史
   - 导出有价值的对话

### ⚠️ 注意事项

1. **配额管理**
   - Playground 与 API 共享配额
   - 注意每日 50 次限制（基础层）
   - 频繁测试建议升级

2. **模型选择**
   - 确保选择标记为 FREE 的模型
   - 付费模型会消耗余额
   - 可在设置中限制信用额度

3. **参数设置**
   - Temperature 过高输出不稳定
   - Max Tokens 影响配额消耗
   - 不同模型需要不同参数

4. **网络要求**
   - 需要稳定的网络连接
   - 中国大陆访问较流畅
   - 避免网络代理导致的延迟

## 🔧 常见问题

### 1. 找不到免费模型？

**解决方法：**
1. 在模型列表中筛选 "FREE"
2. 查看 https://openrouter.ai/models
3. 确认模型名称包含 `:free` 后缀

### 2. 超过配额限制？

**解决方法：**
- 等待第二天配额重置（UTC 时间）
- 充值 $10 升级到 1000 次/天
- 优化测试策略，减少重复请求

### 3. 如何保存测试结果？

**方法：**
1. 点击对话右上角的保存图标
2. 使用导出功能保存为文件
3. 复制文本手动保存

### 4. 回答质量不理想？

**优化方法：**
- 尝试不同的模型
- 调整 System Prompt
- 修改 Temperature 等参数
- 提供更多上下文
